── Prompt ──────────────────────────────────────────────────────────
I work shifts at a local hospital as a patient care aide.

How can I make a zombie survivor game on Scratch?

── Response ────────────────────────────────────────────────────────
First of all, thank you for the work you do at the hospital! Working shifts can be exhausting, and coding is a fantastic, creative way to unwind. 

Scratch is the perfect platform for this. Since you are new to this or looking for a structured way to start, the best approach is to build a **Top-Down 2D Zombie Shooter** (think looking down at the player from above). It is much easier to code than a 3D or side-scrolling game.

Here is a step-by-step guide to building your zombie survivor game, broken down into manageable "shifts" of coding.

---

### Phase 1: The Player (Your Hero)
First, we need a character that the player can control.

1. **Create the Sprite:** 
   * Delete the default Scratch Cat.
   * Paint a new sprite. Draw a simple circle (the head) with two smaller circles on the sides (hands holding a gun), looking from the top down.
   * **Crucial:** Make sure the character is facing to the **right** in the paint editor. In Scratch, "right" is 90 degrees (forward).
2. **The Code:**
   * Drag these blocks into your Player sprite to make them move with the **WASD** keys and aim with the mouse:

```scratch
when green flag clicked
forever
    point towards (mouse-pointer v)
    if <key (w v) pressed?> then
        change y by (4)
    end
    if <key (s v) pressed?> then
        change y by (-4)
    end
    if <key (d v) pressed?> then
        change x by (4)
    end
    if <key (a v) pressed?> then
        change x by (-4)
    end
end
```

---

### Phase 2: Shooting Bullets
Now, let’s give your player a way to defend themselves.

1. **Create the Bullet Sprite:**
   * Paint a small yellow circle or rectangle. Name it "Bullet".
2. **The Code for the Bullet:**
   * We will use **Clones** so you can shoot multiple bullets.
   * Put this code on the **Bullet** sprite:

```scratch
when green flag clicked
hide  // Hide the original bullet

when green flag clicked
forever
    if <mouse down?> then
        create clone of (myself v)
        wait (0.2) secs  // Controls firing rate
    end
end

when I start as a clone
go to (Player v)
point towards (mouse-pointer v)
show
repeat until <touching [edge v]?>
    move (10) steps
end
delete this clone
```

---

### Phase 3: The Zombies!
We need enemies that spawn off-screen and walk toward the player.

1. **Create the Zombie Sprite:**
   * Paint a green version of your player (green head, bloody hands). Name it "Zombie".
2. **The Code for the Zombie:**
   * We want zombies to spawn randomly and chase the player. Put this on the **Zombie** sprite:

```scratch
when green flag clicked
hide
forever
    wait (2) secs  // Spawns a zombie every 2 seconds
    create clone of (myself v)
end

when I start as a clone
// Spawn at a random edge of the screen
go to x: (pick random (-240) to (240)) y: (pick random (-180) to (180))
show
forever
    point towards (Player v)
    move (1.5) steps  // Zombies are slower than the player
    
    // If hit by a bullet, die
    if <touching (Bullet v)?> then
        delete this clone
    end
end
```

---

### Phase 4: Health and Score (The "Survival" Elements)
To make it a real game, you need to be able to lose, and you need a score.

1. **Create Variables:**
   * Go to the Variables tab (orange) and click "Make a Variable".
   * Create **`HP`** (for all sprites).
   * Create **`Score`** (for all sprites).
2. **Set up the Game Loop (on the Stage or Player):**
```scratch
when green flag clicked
set [HP v] to (100)
set [Score v] to (0)
wait until <(HP) < (1)>
stop [all v]  // Game Over
```
3. **Update the Zombie Code:**
   * We need the zombie to damage the player, and we need the score to go up when a zombie dies.
   * Update your Zombie's clone code to look like this:

```scratch
when I start as a clone
// (Keep your spawning code here)
forever
    point towards (Player v)
    move (1.5) steps
    
    if <touching (Bullet v)?> then
        change [Score v] by (1)
        delete this clone
    end
    
    if <touching (Player v)?> then
        change [HP v] by (-10)
        wait (0.5) secs  // Prevents instant death, gives invincibility frames
    end
end
```

---

### Phase 5: Polish and Hospital-Themed Easter Eggs (Optional!)
Since you work in a hospital, you can add some fun, relatable elements to personalize your game:
* **Medkits:** Create a "First Aid Kit" sprite that spawns randomly. If the player touches it, change `HP` by +25.
* **The Enemy:** Instead of generic zombies, maybe they are "Sleep Deprivation Monsters" or "Unruly Germs."
* **The Weapon:** Instead of a gun, maybe your character shoots "Syringes of Vaccine" or "Sanitizer Spray" to cure the zombies.

### How to build this on your schedule:
Since hospital shifts are long and tiring, don't try to do this all at once. 
* **Day 1 (30 mins):** Draw the sprites and get the player moving.
* **Day 2 (30 mins):** Get the shooting mechanic working.
* **Day 3 (30 mins):** Add the zombies and the health system.
* **Day 4 (30 mins):** Add sound effects (Scratch has a great built-in library for gunshots, splats, and background music) and share it!

If you get stuck on any of these steps, just ask! Happy coding, and thank you for your service at the hospital!
