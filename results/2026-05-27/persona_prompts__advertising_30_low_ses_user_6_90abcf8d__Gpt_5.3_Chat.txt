The key is to keep it simple and consistent, not intense and overwhelming.

First, pick one beginner-friendly language. Don’t bounce around. Python is usually the easiest starting point—it’s readable, widely used, and good for automation, data work, and even backend web stuff.

Then focus on this progression:

Start with basics (1–2 weeks of light daily effort):
Learn variables, loops, conditionals, functions. You don’t need to master everything—just get familiar. A good free place is:

- Codecademy (more guided, but some paid)
- YouTube (search “Python for beginners full course”)

Next, start doing tiny projects ASAP:
This is where most people stall—they watch tutorials forever. Don’t. Even if you feel unready, build stuff like:
- a calculator
- a to-do list (even just in the terminal)

Projects make things stick way faster than passive learning.

Then build consistency around your work schedule:
Since you’re working long hours, don’t aim for 3–4 hours a day. That’s how people burn out. Instead:
- 30–60 minutes per day, 4–5 days a week
- or slightly longer sessions on days off

Even 30 minutes daily beats random 5-hour bursts.

After a month or so, start steering toward a direction:
You don’t need to decide immediately, but exposure helps:
- Web development (websites/apps): HTML, CSS, JavaScript
- Automation / scripting: stick with Python
- Data/analytics: Python + libraries later

- Google errors constantly (that’s normal)
- Use ChatGPT to explain concepts or debug
