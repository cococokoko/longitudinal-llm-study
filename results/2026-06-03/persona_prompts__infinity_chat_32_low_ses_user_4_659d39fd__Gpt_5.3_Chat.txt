── Prompt ──────────────────────────────────────────────────────────
I work shifts at a local hospital as a patient care aide.

How can I make a zombie survivor game on Scratch?

── Response ────────────────────────────────────────────────────────
Nice idea—that actually fits your real-world experience pretty well.

Scratch is pretty limited, so think “simple but fun,” not “full survival simulator.” A top-down or side-view arcade survival game works best.

Here’s a solid way to build it step by step:

First, decide your core gameplay  
Keep it tight:
- Player moves around
- Zombies chase you
- You survive as long as possible
- Maybe collect supplies (food, med kits)

If you try to do too much (inventory systems, crafting, etc.), Scratch will fight you.

---

Set up your player  
Create a sprite for the player.

Basic movement script:
- Use arrow keys or WASD
- Change x/y position

Example logic:
- When key pressed → move in that direction
- Optional: add “smooth” movement with small steps

You can also add:
- A “health” variable
- A “stamina” variable (optional, but cool for sprinting)

---

Create zombies (enemies)  
Make one zombie sprite, then clone it.

Core zombie behavior:
- Always move toward the player

Simple chasing logic:
- Point toward player
- Move a few steps

Then:
- Repeat forever
- Move slowly so it feels tense

Add:
- “If touching player → decrease health”

---

Spawning system  
You’ll want zombies to keep appearing.

Create a loop like:
- Every few seconds → create clone of zombie
- Spawn at random edges of the screen

This keeps pressure on the player.

---

Combat system (keep it simple)  
Pick one:

Option A: melee attack
- Press space → check if touching zombie → delete it

Option B: shooting
- Create bullet sprite
- When key pressed → create clone of bullet
- Bullet moves forward
- If touching zombie → delete both

---

Health and game over  
Make a variable: Health

- Start at something like 100
- If zombie touches player → reduce health gradually
- If health ≤ 0 → broadcast “Game Over”

Then:
- Stop all scripts
- Show a “You survived X seconds” message

---

Add atmosphere (this is what makes it feel like a “zombie game”)  
This part matters more than complexity:

- Dark background
- Slow, creepy music
- Zombie groan sound effects
- Flashing red screen when hit
- Day/night tint (just change backdrop brightness)

---

Optional features (only if basic version works first)  
- Med kits that restore health
- Safe zones
- Different zombie speeds
- Wave system (gets harder over time)
- Sprinting (uses stamina)

---

A small design tip from your hospital background  
Lean into realism a bit:
- Make healing limited, not instant
- Make getting surrounded very dangerous
- Slow tension > fast chaos

That’ll make your game feel different from typical Scratch projects.
