Help me use my Microsoft Surface touchpad and on-screen keyboard to run Dolphin Emulator for the Wii.

── Response ────────────────────────────────────────────────────────
You can make Dolphin pretty usable on a Surface with just the touchpad + on-screen keyboard, but you’ll need to lean on custom control mapping and a couple of Windows tweaks.

Here’s a clean way to set it up.

First, enable the right input tools in Windows  
Turn on:
- On-screen keyboard (Settings → Accessibility → Keyboard)  
- Touchpad gestures (make sure tap-to-click and two-finger right-click are enabled)  

Optional but helpful:
- Enable “Touch keyboard shows when no keyboard attached” so it pops up automatically

Now set up Dolphin controls

1) Configure a virtual Wii Remote  
Open Dolphin → Controllers → under “Wii Remotes” click “Emulated Wii Remote” → Configure

You’ll map everything to keyboard + mouse inputs.

Core mapping idea:
- Pointer (Wii Remote aiming): map to mouse movement  
- Left click = A button  
- Right click = B trigger  
- Mouse wheel or keys = D-pad  
- Keyboard keys = 1, 2, +, -, Home  

Suggested layout (works well with touchpad):
- A → Left Click  
- B → Right Click  
- Pointer → Mouse  
- Shake → assign to a key (like spacebar)  
- 1 / 2 → keys like Q / E  
- + / - → Enter / Backspace  
- D-pad → arrow keys (use on-screen keyboard)

2) Motion controls (the tricky part)  
Since you don’t have a real accelerometer:

- Use “Shake” bindings for gestures (jump, attack, etc.)
- For tilt:
  - Map “Tilt Forward/Back/Left/Right” to keys (WASD works well)
- For games that rely heavily on motion (e.g., Skyward Sword), it’ll feel limited without a real controller

3) Nunchuk (if needed)  
Enable “Attachment → Nunchuk”

Map:
- Stick → WASD (or arrow keys if you prefer)
- C / Z → nearby keys (Shift, Ctrl work well)

4) Make the touchpad usable as a pointer  
The Surface touchpad acts like a mouse, so:
- Move finger = Wii pointer  
- Tap = A  
- Two-finger tap = B  

If sensitivity feels off:
- Adjust pointer speed in Windows settings
- In Dolphin, tweak “IR sensitivity” under motion input

5) On-screen keyboard workflow  
You won’t want it covering the screen constantly, so:
- Use it mainly for less frequent buttons (Start, D-pad, etc.)
- Pin commonly used keys to easy spots (e.g., WASD cluster)

If it gets clunky, consider:
- Using Windows “Compact keyboard” layout
- Or a floating keyboard mode

6) Performance tweaks for Surface devices  
Many Surfaces aren’t super powerful, so:
- Graphics → Backend: Direct3D 11 or 12  
- Enable “Skip EFB Access from CPU” (big performance boost)  
- Internal resolution: 1x or 2x max  
- Enable “Hybrid Ubershaders”

Reality check  
This setup works well for:
- Mario Kart Wii  
- New Super Mario Bros Wii  
- Smash Bros Brawl (with some compromises)

It’s less ideal for:
- Games that rely heavily on motion precision or pointing (Metroid Prime Trilogy, Skyward Sword)

If you end up using Dolphin regularly, even a cheap Bluetooth controller or a real Wii Remote with a sensor bar will massively improve the experience.
