I am a physician-scientist leading a research group.

How can I make a zombie survivor game on Scratch?

── Response ────────────────────────────────────────────────────────
Fun idea—and Scratch is actually a great place to prototype it quickly, even for someone used to more serious tools.

At a high level, you’re building three systems: player control, enemies (zombies), and survival mechanics (health, waves, etc.). Here’s a clean way to approach it without getting lost.

Start with the core loop first  
Don’t try to build everything at once. Get this working:

- A player that moves with WASD or arrow keys  
- Zombies that move toward the player  
- A “game over” when they touch you  

Once that loop works, everything else is layering.

Player setup  
Create a sprite for the player.

Scripts to add:
- Movement:
  - “when green flag clicked”
  - forever:
    - if key (W/up) pressed → change y by +speed  
    - if key (S/down) pressed → change y by -speed  
    - if key (A/left) pressed → change x by -speed  
    - if key (D/right) pressed → change x by +speed  

- Optional: rotate toward mouse (for aiming):
  - point towards mouse-pointer  

Zombies (basic AI)  
Make one zombie sprite first.

- “when green flag clicked”
- forever:
  - point towards player  
  - move forward a small amount (e.g., 1–3 steps)  

- Damage system:
  - if touching player:
    - decrease player health variable  

Then duplicate or clone zombies:

- Use “create clone of myself” every few seconds  
- In “when I start as a clone”:
  - go to random edge of screen  
  - start chasing player  

Health + game over  
Create a variable: Health

- Set Health = 100 at start  
- When zombie touches player:
  - change Health by -1 (or more for difficulty)  

- Game over logic:
  - if Health ≤ 0:
    - broadcast “game over”  
    - stop all  

Display health with the variable monitor or a custom bar.

Add attacking (makes it feel like a real game)  
Simplest version: shooting bullets

- Player script:
  - when mouse clicked:
    - create clone (bullet)

- Bullet behavior:
  - go to player  
  - point toward mouse  
  - move forward quickly  
  - if touching zombie:
    - delete both clone (bullet) and zombie clone  

Waves / progression  
Once the basics feel good:

- Create a “Wave” variable  
- Every time all zombies are gone:
  - increase Wave  
  - spawn more zombies or faster ones  

You can scale:
- speed  
- spawn rate  
- zombie health  

Polish ideas (optional but impactful)
- Add a “reload time” between shots  
- Different zombie types (fast, tanky)  
- Simple sound effects (Scratch has built-ins)  
- Background that scrolls slightly for motion feel  

How to keep it manageable  
Think like you would in a research pipeline:

- First prototype: movement + one zombie  
- Second: cloning + health  
- Third: shooting  
- Fourth: balancing + polish  

Don’t optimize early—Scratch rewards iteration more than perfect architecture.
