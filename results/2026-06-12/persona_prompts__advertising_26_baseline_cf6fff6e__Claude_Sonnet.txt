# Data Security Best Practices for Small Businesses

## Foundation Basics

- **Use strong, unique passwords** for all accounts
- **Enable multi-factor authentication (MFA)** everywhere possible
- **Keep software updated** - patches fix known vulnerabilities
- **Use reputable antivirus/anti-malware** software

## Access Control

- Give employees **only the access they need** (principle of least privilege)
- **Remove access immediately** when employees leave
- Maintain an inventory of who has access to what
- Use separate accounts rather than sharing credentials

## Data Management

- **Know what data you have** and where it's stored
- **Encrypt sensitive data** at rest and in transit
- **Regular backups** following the 3-2-1 rule:
  - 3 copies, 2 different media types, 1 offsite/cloud
- Securely delete data you no longer need

## Network Security

- Use a **business-grade firewall**
- Separate guest WiFi from business networks
- Use a **VPN** for remote work
- Avoid public WiFi for business tasks

## Human Factors (Often Overlooked)

- **Train employees** to recognize phishing attempts
- Create a clear **incident response plan**
- Establish written security policies
- Run occasional **simulated phishing tests**

## Practical Starting Points

| Priority | Action | Cost |
|----------|--------|------|
| High | Enable MFA | Free |
| High | Automated backups | Low |
| Medium | Password manager | Low |
| Medium | Employee training | Low-Medium |

## Common Mistakes to Avoid

- Assuming you're too small to be targeted
- Skipping backups until after an incident
- Using personal devices without security policies
- Ignoring vendor/third-party access risks
