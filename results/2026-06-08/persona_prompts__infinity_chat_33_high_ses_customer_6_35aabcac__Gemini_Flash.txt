Help me use my Microsoft Surface touchpad and on-screen keyboard to run Dolphin Emulator for the Wii.

── Response ────────────────────────────────────────────────────────
Running Dolphin Emulator (for Wii) on a Microsoft Surface using only the touchpad and the Windows On-Screen Keyboard (OSK) is entirely possible, though it requires a specific configuration to bypass the physical limitations of touch inputs. 

Below is a streamlined, step-by-step guide to configuring this setup for optimal playability, followed by a highly recommended "efficiency upgrade" that will significantly improve your experience.

---

### Phase 1: Essential Prerequisites
To make this setup work, you must run Dolphin in **Windowed Mode** (not Fullscreen). If Dolphin is in fullscreen, the Windows On-Screen Keyboard will be hidden behind the game window.

1. Open Dolphin.
2. Go to **Options > Graphics Settings**.
3. Under the **General** tab, ensure **Start in Fullscreen** is **unchecked**.
4. Open your Windows On-Screen Keyboard:
   * Press `Win + Ctrl + O` to launch the classic **On-Screen Keyboard (OSK)**. 
   * *Note: Use the classic OSK (osk.exe) rather than the modern Windows Touch Keyboard, as the classic version can be resized easily and stays pinned on top of other windows more reliably.*

---

### Phase 2: Configuring Dolphin for Touchpad & Keyboard
You need to map the Wii Remote’s motion controls (the IR pointer) to your touchpad, and the buttons to your keyboard.

1. In Dolphin, click **Controllers** on the main toolbar.
2. Under **Wii Remotes**, set **Wii Remote 1** to **Emulated Wii Remote**, then click **Configure**.
3. At the top left of the configuration window, set the **Device** to **Keyboard/Mouse** (it may appear as `WGInput/0/Keyboard Mouse`).

#### Step 3a: Mapping the Touchpad (Wii Pointer)
The Wii Remote requires a pointer. We will map this to your Surface touchpad.
1. In the controller configuration window, go to the **Motion Simulation** tab.
2. Look for the **Point** section.
3. Right-click the box next to **Up** and select `Cursor Y-` (or pull the cursor up).
4. Do the same for the others:
   * **Down**: `Cursor Y+`
   * **Left**: `Cursor X-`
   * **Right**: `Cursor X+`
5. *Result:* Moving your finger on the touchpad will now move the Wii pointer on screen.

#### Step 3b: Mapping the Buttons to the OSK
Since you are using an on-screen keyboard, you want to map the most critical Wii buttons to keys that are physically close to each other on the OSK to minimize hand movement.

Go to the **General and Buttons** tab in the configuration window:
* **A Button** (Primary action): Map to `Spacebar` or `Left Click` (Touchpad tap).
* **B Button** (Trigger/Secondary): Map to `Right Click` or `Shift`.
* **D-Pad** (Movement): Map to `W` (Up), `A` (Left), `S` (Down), `D` (Right).
* **Minus / Plus**: Map to `1` and `2`.

*Click **Save** at the top right and name this profile "Surface_Touch".*
